# 2026-05-23

**Noto Sans Japanese**は Google と Adobe が共同で開発したオープンソースの日本語フォントで、デファクトスタンダード

今日は、Tailwind CSS での`フォント`の適用方法と競技プログラミングにおける`ソートアルゴリズム`について勉強した。

## まず、Tailwind CSSでのフォントの適用方法はすごい簡単！
手順は

**1. `@theme`**で設定内容の確認

CSSファイルで以下のようにフォントを定義

```CSS
@theme {
  --font-fraunces: "Fraunces", serif;
  --font-shippori: "Shippori Mincho", serif;
  --font-sans-jp: "Noto Sans JP", sans-serif;
}
```
**2. ReactのclassNameにfont-sans-jp**

```html
<h1 class="font-sans-jp bg-blue-600 text-white px-4 py-2 rounded">
	タイトル
</h1>
```

##### オプション（太さの指定）

フォントを適用したうえで、太さを変えたいときは、Tailwindひゅじゅんの`font-bold`や`font-medium`を組み合わせて使える！


## ソートアルゴルズムについて


ソートアルゴリズムは「ただ、データを照準に並べる」だけではない！
アルゴリズムの効率性を表す「O記法」を学べる

- バブルソートや選択ソート：　直感的で書きやすいけれど、データ量が2倍になると計算時間も４倍になる （O（N^2））
- クイックソートやマージソート：仕組みは複雑だけれど、データ量が2倍になっても、計算時間は2倍ちょっとで済む（O（n log n））
