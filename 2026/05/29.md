## JavaScript で変数宣言 var が使われなくなった理由
---

### 理由1： 変数の巻き上げ

JavaScript には、コードを実行する前に、「宣言だけが先頭に移動して、中身はundefined で初期化」されます。
なので、以下のコードのように、まだ宣言されていない変数にアクセスすると、undefinedと表示されてしまいます。

```JavaScript
console.log(magic) // エラーにならずに、 undefined と出力される

var magic = "手品";

console.log("手品") // ここで「手品」が出力
```


### 理由2： 同じ名前の変数を再宣言できてしまう

大きなプログラムを書いているときに、すでに使っている変数名を、もう一度`var`でうっかり宣言してしまうと、前のデータを上書きしてしまいます。

`const` と `let` だと、`Identifier 'userName' has already been declared`と、エラーを出して止めてくれます。

### 理由3： ブロックスコープを無視する

`if`や`for`文などの`{}`ブロックを突き抜けて外側まで影響を与えてしまいます。

```JavaScript
if (true) {
  var framework = "React";
}

// ifブロックの外なのに、変数にアクセスできてしまう
console.log(framework) // 出力: React
```

## 非制御コンポーネントとは

**非制御コンポーネント**とはフォームの入力値を`React`の`useState`で管理するのではなく、‘DOM‘で扱うコンポーネントのことを言います。

メリット: 

- useStateを経由して保存していないため、入力ごとに再レンダリングが発生しない

デメリット:

- React を返していないため、入力値ごとのバリデーションができない。

##### RHFとの関係

React Hook Form は非制御コンポーネントを採用していて、`ref`属性に`register`を渡すことで、フォームの実装ができる。
