## 学んだこと

### useQueryの責務
---
useQueryがやるのは**フェッチ十キャッシュ管理**、「データが取れたタイミングで初期値をセット」は useQuery ぼ責務外。　そのため、useEffectを使い、resetに通知してあげる必要がある。

```typescript
useEfffect(() => {
  if (!book) return
  reset({...book})
}, [book, reset])
```

**これは TanStack Query V5 公式が推奨する react-hook-formとの統合パターン**

---
### useQueryのオプション、enabledとは？
---
`enabled`オプションはデータ取得の自動実行を制御するために使用される。
`enabled`の値が true にならない限り`queryFn`は実行されない。
もし、存在しない値であっても実行されないので、`enabled`がないときと比べて「キャッシュが汚れない」という利点がある。

### git rebase コマンド
---
ブランチの分岐点を別の場所に移動させるコマンドです。

`feature`ブランチで作業するはずが、`main`ブランチで作業をしてしまった際に、
`feature`ブランチへ移動し`git rebase main`うぃ実行すると、「最初からfeatureブランチで作業していた」状態になる
