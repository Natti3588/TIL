# 2026-04-02

## 学んだこと

チーム開発を最近行っていなかったので復習として、GitのWorkFlowを勉強した。

- `git fetch` はリモートの最新情報を取得するだけで、作業ブランチには影響しない
- `git pull` = `git fetch` + `git merge`
- `git fetch` 後に `git log main..origin/main` や `git diff` で変更内容を確認してからマージできる
- `origin` はリモートリポジトリURLのエイリアス（`git clone` で自動設定される）
- プルリクエストの流れ: ブランチ作成 → コミット → push → PR作成 → レビュー → マージ

## わかったこと・気づき

- `git fetch` は「確認してから判断できる」安全なコマンド
- `git pull` でいきなりマージするより、`fetch` → 確認 → `merge` の流れが安全
- TILリポジトリをGitHubにpublicで公開すると学習意欲のアピールにもなる

## まだわからないこと

- コンフリクトが起きたときの具体的な対処法
- レビューの具体的なやり方・観点
