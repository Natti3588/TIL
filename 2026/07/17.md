# SPAとは何か？

SPAは基本的にサーバーから返ってくるのは最初の1枚のHTMLだけで、あとはJSがゴリゴリDOMを書き換えて画面を作ってるだけ。
ということは、ブラウザは「URLごとに違うページがある」という認識がないから、実質ずっと同じ1ページを見せられ続けてるだけ。

でも実際のアプリだと `/users/1` と `/users/2` で違う中身を見せたいし、リロードしても同じ画面が出てほしいし、URLをコピペして人に共有したりもする。そこで出てくるのが React Router みたいなルーティングライブラリ。

やってることは、`pushState` とか `popstate` を使ってアドレスバーの見た目だけを書き換えつつ、今のURLに応じてどのコンポーネントを出すかをJS側で自前で振り分けてる、という感じらしい。裏で毎回サーバーにリクエストが飛んでるわけじゃないので、画面遷移が速く感じるのはそのため。

SPAのURLに応じたページ遷移はブラウザ標準の機能じゃなくて、JS側が擬似的に再現している。
